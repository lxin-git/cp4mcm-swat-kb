If you have completed the above prerequisite steps without issue you can ignore this
section.

1. If you are experiencing SSL errors when trying to access CP4I, OpenShift Cluster Web
   UI's or the Aspera Test Cluster bookmark:

   As the environment has been in a shut down state the necessary certificate rotation may
   not have taken place. To manually rotate certificates:

   1. List any outstanding CSR's by opening a Terminal and running:

      `./Documents/scripts/check-csr.sh`

      If you see 'no resources found' you do not have any outstanding CSR's.

   1. Approve any outstanding CSR's

      `./Documents/scripts/approve-csr.sh`

      Let the CSR's approve and give the cluster a few minutes before trying the Web UI
      again.
